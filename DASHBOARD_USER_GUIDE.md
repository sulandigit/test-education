# 领课教育用户数据展示大屏系统 - 使用说明

## 1. 系统介绍

领课教育用户数据展示大屏系统是基于现有领课教育平台开发的实时数据可视化系统，为管理层和运营团队提供直观的数据洞察和实时监控能力。

## 2. 功能特性

### 2.1 实时数据监控
- **用户统计**: 在线用户数、今日新增用户、用户总数
- **课程数据**: 热门课程排行、视频播放次数、课程完成率
- **收入分析**: 今日收入、本月收入、支付方式分布
- **系统状态**: 服务健康状态、系统负载、异常监控

### 2.2 数据可视化
- **实时图表**: 基于ECharts的动态图表展示
- **地域分布**: 用户地理位置分布地图
- **趋势分析**: 多维度数据趋势对比
- **响应式设计**: 适配不同屏幕尺寸

### 2.3 实时推送
- **WebSocket连接**: 实时数据推送
- **自动刷新**: 数据自动更新机制
- **异常告警**: 系统异常实时通知
- **连接状态**: 实时连接状态显示

## 4. 操作指南

### 4.1 访问大屏

1. 打开浏览器，访问大屏地址：`http://your-domain.com/dashboard-ui/`
2. 系统会自动加载并建立WebSocket连接
3. 数据加载完成后显示实时数据大屏

### 4.2 数据解读

#### 用户统计区域
- **当前在线**: 显示当前正在使用平台的用户数量
- **今日新增**: 当日新注册的用户数量
- **用户总数**: 平台注册用户总数

#### 核心业务指标
- **今日收入**: 当日订单收入总额
- **本月收入**: 本月累计收入总额
- **今日订单**: 当日订单数量
- **视频播放**: 当日视频播放次数

#### 热门课程排行
- 按学习人数或收入排序的TOP10课程
- 显示课程名称、学习人数、收入金额
- 实时更新课程热度变化

#### 收入趋势分析
- 显示收入随时间的变化趋势
- 支持多时间维度查看（小时、天、周）
- 对比分析功能

#### 用户地域分布
- 饼图显示用户按省份/城市的分布
- 显示具体用户数量和占比
- 支持点击查看详细信息

#### 支付方式分布
- 微信支付、支付宝支付等方式的占比
- 显示订单数量和金额分布
- 实时更新支付数据

#### 系统状态监控
- 在线服务数量和健康状态
- 系统负载和异常数量
- 数据库连接状态

### 4.3 交互功能

#### 图表交互
- **悬停显示**: 鼠标悬停在图表上显示详细数据
- **图例控制**: 点击图例显示/隐藏数据系列
- **缩放功能**: 支持图表缩放和平移操作

#### 实时更新
- **自动刷新**: 数据每5秒自动更新
- **连接状态**: 右上角显示WebSocket连接状态
- **更新时间**: 底部显示最后更新时间

#### 响应式适配
- **大屏模式**: 1920x1080及以上分辨率优化显示
- **中屏适配**: 1366-1920px分辨率自动调整布局
- **小屏适配**: 1366px以下显示关键指标

## 5. 数据更新机制

### 5.1 更新频率
- **实时数据**: 5秒更新一次
- **用户统计**: 1分钟更新一次
- **课程数据**: 5分钟更新一次
- **收入数据**: 1分钟更新一次
- **系统状态**: 30秒更新一次

### 5.2 缓存策略
- **热点数据**: Redis缓存，过期时间5-10分钟
- **列表数据**: 内存缓存，过期时间5分钟
- **趋势数据**: 数据库缓存，过期时间30分钟

## 6. 常见问题

### 6.1 页面显示问题

**问题**: 页面加载缓慢或空白
**解决**: 
- 检查网络连接是否正常
- 清除浏览器缓存后重新访问
- 确认服务器状态是否正常

**问题**: 数据显示为0或空
**解决**:
- 检查后端服务是否正常运行
- 确认数据库连接是否正常
- 查看控制台是否有错误信息

### 6.2 实时更新问题

**问题**: 数据不实时更新
**解决**:
- 检查WebSocket连接状态（右上角指示器）
- 刷新页面重新建立连接
- 检查网络防火墙设置

**问题**: 连接频繁断开
**解决**:
- 检查网络稳定性
- 确认服务器负载是否过高
- 联系系统管理员检查服务状态

### 6.3 性能问题

**问题**: 图表渲染卡顿
**解决**:
- 使用高性能浏览器（Chrome、Firefox）
- 关闭其他占用资源的程序
- 降低浏览器缩放比例

**问题**: 内存占用过高
**解决**:
- 定期刷新页面释放内存
- 关闭其他浏览器标签页
- 升级浏览器到最新版本

## 7. 最佳实践

### 7.1 显示设备推荐
- **屏幕尺寸**: 55寸及以上
- **分辨率**: 1920x1080或4K
- **刷新率**: 60Hz及以上
- **亮度**: 根据环境光线调整

### 7.2 浏览器设置
- 建议使用Chrome或Firefox浏览器
- 启用硬件加速功能
- 禁用浏览器自动休眠
- 设置全屏显示模式

### 7.3 网络要求
- 带宽: 10Mbps及以上
- 延迟: 50ms以下
- 稳定性: 99%以上
- 支持WebSocket协议

## 8. 技术规格

### 8.1 前端技术
- **框架**: Vue.js 3.x
- **图表库**: ECharts 5.x
- **通信**: WebSocket + HTTP
- **样式**: CSS3 + 响应式设计

### 8.2 后端技术
- **框架**: Spring Boot 2.6.x
- **微服务**: Spring Cloud Alibaba
- **缓存**: Redis
- **数据库**: MySQL 5.7+

### 8.3 浏览器兼容性
- Chrome 80+
- Firefox 75+
- Safari 13+
- Edge 80+
